# Git 初始化并推送到 GitHub 笔记

## 1. 初始化本地 Git 仓库

进入项目目录：

```
cd ~/Desktop/study/git\ test
```

初始化：

```
git init
```

结果：

```
Initialized empty Git repository
```

说明当前目录已经变成 Git 仓库。

------

# 2. 配置 Git 用户信息

第一次提交时：

```
git commit -m "first commit"
```

出现：

```
Author identity unknown

Please tell me who you are.
```

原因：

Git 不知道提交者是谁，需要配置用户名和邮箱。

配置：

```
git config --global user.name "zhipengjun"

git config --global user.email "zhipengjun521@gmail.com"
```

查看：

```
git config --global --list
```

输出：

```
user.name=zhipengjun
user.email=zhipengjun521@gmail.com
```

# 4. 创建第一次提交

提交：

```
git commit -m "first commit"
```

如果出现：

```
nothing to commit
```

说明：

当前没有文件进入暂存区。

正确流程：

```
echo hello > 1.txt

git add .

git commit -m "add 1.txt"
```

成功：

```
[root-commit c8b8070]
create mode 100644 1.txt
```

# 8. 第一次 push 失败原因

执行：

```
git push -u origin main
```

错误：

```
error: src refspec main does not match any
```

原因：

本地没有 commit。

检查：

```
git log
```

出现：

```
fatal: your current branch 'main' does not have any commits yet
```

解决：

```
git add .

git commit -m "first commit"
```

再 push。

------

# 9. GitHub 网络连接问题

错误：

```
Failed to connect to github.com:443
```

测试：

```
ping github.com
```

结果：

```
20.205.243.166
```

说明：

DNS 和网络正常。

但是：

HTTPS 443 访问失败。

解决：

配置代理：

```
git config --global http.proxy http://127.0.0.1:7897

git config --global https.proxy http://127.0.0.1:7897
```

查看：

```
git config --global --list
```

应该出现：

```
http.proxy=http://127.0.0.1:7897
https.proxy=http://127.0.0.1:7897
```

------

# 10. push 被拒绝 fetch first

错误：

```
! [rejected] main -> main (fetch first)
```

原因：

GitHub 仓库已经存在提交。

例如：

GitHub 创建仓库时勾选：

```
README.md
.gitignore
LICENSE
```

远程：

```
A
 \
  B
```

本地：

```
C
```

两个历史不一致。

------

# 11. 解决远程和本地合并

使用 rebase：

```
git pull origin main --rebase
```

成功：

```
Successfully rebased and updated refs/heads/main.
```

然后：

```
git push -u origin main
```

成功：

```
main -> main

branch 'main' set up to track 'origin/main'
```
