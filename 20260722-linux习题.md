# 一、Linux 命令题

## 1. 删除目录及目录内所有内容

命令：

```
rm -rf dir
```

解释：

```
-r
```

递归删除：

- 删除目录
- 删除目录里面的所有文件和子目录

```
-f
```

强制删除：

- 不提示确认
- 直接执行删除

------

## 2. 统计并去重显示 2.txt 内容（需先排序）

命令：

```
sort 2.txt | uniq
```

如果需要统计重复次数：

```
sort 2.txt | uniq -c
```

解释：

流程：

```
2.txt
  |
  ↓
sort 排序
  |
  ↓
uniq 去重/统计
```

注意：

`uniq` 只能处理连续重复内容，所以必须先 `sort`。

# 二、Xshell 远程连接服务器必备 4 个要素

使用 Xshell SSH 连接服务器，需要：

## 1. 服务器 IP 地址

例如：

```
192.168.1.100
```

用于定位服务器。

## 2. SSH 服务开启

服务器需要运行：

```
sshd
```

查看：

```
systemctl status sshd
```

## 3. SSH 端口

默认：

```
22
```

如果修改过，需要填写对应端口。

## 4. 用户名和密码（或密钥）

例如：

```
root
密码
```

或者：

```
SSH Key
```

总结：

```
IP地址
+
SSH服务
+
端口
+
用户名密码
```

# 三、无法远程连接服务器排查思路

排查顺序：

------

## 第一步：检查网络是否连通

测试：

```
ping 服务器IP
```

例如：

```
ping 192.168.1.100
```

检查：

- 网络是否正常
- IP 是否正确

------

## 第二步：检查 SSH 服务是否启动

服务器：

```
systemctl status sshd
```

启动：

```
systemctl start sshd
```

------

## 第三步：检查端口是否开放

查看：

```
ss -lntp | grep 22
```

检查：

- 防火墙
- 云服务器安全组
- 端口映射

------

## 第四步：检查用户名密码

确认：

- 用户存在
- 密码正确
- 是否禁止 root 登录

------

总结：

```
1. 网络
2. SSH服务
3. 端口、防火墙
4. 用户认证
```

# 四、vim 出现 .swp 文件原因及处理

## 原因：

`.swp` 是 vim 的交换文件。

产生原因：

- 编辑文件时异常退出
- 多人同时编辑同一个文件
- vim 未正常保存退出

例如：

编辑：

```
vim test.txt
```

产生：

```
.test.txt.swp
```

------

## 处理方式

打开文件：

```
vim test.txt
```

出现：

```
Found a swap file
```

------

## 1. 想恢复未保存内容

选择：

```
r
```

或者：

```
vim -r test.txt
```

恢复交换文件内容。

------

## 2. 不需要未保存内容

删除交换文件：

```
rm -f .test.txt.swp
```

重新编辑：

```
vim test.txt
```

------

总结：

| 需求           | 操作           |
| -------------- | -------------- |
| 恢复未保存内容 | vim -r 文件    |
| 不要未保存内容 | 删除 .swp 文件 |

# 五、CentOS/Kylin 修改网卡 IP 后如何生效

修改网卡配置后，需要重新加载网络。

两种方法：

------

## 方法1：重启网络服务

CentOS 7：

```
systemctl restart network
```

------

## 方法2：重启网卡

例如：

```
ifdown eth0
ifup eth0
```

或者：

```
nmcli connection reload
nmcli connection up eth0
```

------

## 方法3（最直接）

重启系统：

```
reboot
```

------

考试写：

```
1. systemctl restart network
2. reboot
```

即可。

# 六、防火墙优化策略

## 什么情况下开启？

生产环境：

需要：

- 服务器暴露公网
- 对外提供服务
- 需要安全防护

例如：

```
Web服务器
数据库服务器
线上业务服务器
```

开启防火墙：

```
systemctl start firewalld
```

------

## 什么情况下关闭？

学习、测试环境：

例如：

- 搭建实验环境
- 调试服务
- 多服务联调

关闭：

```
systemctl stop firewalld
```

禁止开机启动：

```
systemctl disable firewalld
```

------

总结：

| 环境         | 防火墙             |
| ------------ | ------------------ |
| 生产服务器   | 开启，开放指定端口 |
| 学习测试环境 | 可关闭，方便调试   |
